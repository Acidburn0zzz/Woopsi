Developer Guide - Creating a Simple GUI

Creating a GUI is divided into four stages - creating a Woopsi object, creating screens, creating windows, and creating gadgets.

  • Creating a Screen
  • Creating a Window
  • Creating a Gadget
  • Summary

Creating a Woopsi Object

Creating a GUI with Woopsi is simple. To begin, we need to create an instance of the Woopsi class. In the "main.cpp" file, in the "main()" function, add the following:

// Create woopsi
Woopsi woopsi;

The "woopsi" object we have created gives us our entry point into the GUI.

Creating a Screen

Next we need to add a new screen so that we have a workspace for the interface. To do this, we use this code:

// Create screen
Screen* screen = woopsi.addScreen("My Screen");

This achieves two things. We've created a new screen within Woopsi and given it the name "My Screen" (this appears in the screen's title bar). We've also created a pointer to the new screen that we'll need in the next step.

Creating a Window

Now that we've got a screen, we need to add windows to it. Windows are added to screens using this function:

Window* Screen::addWindow(s16 x, s16 y, u16 width, u16 height, char* title, bool isBorderless, bool isCloseable)

The parameters are as follows:

xThe x co-ordinate of the top-left corner of the window
yThe y co-ordinate of the top-right corner of the window
widthThe width of the window
heightThe height of the window
titleThe name of the window to be displayed in its title bar
isBorderlessSetting this to "true" disables the blue borders and bevel box around the window
isCloseable"True" enables the close gadget; "False" disables the close gadget

Note that although the x and y co-ordinates are entered as a signed short, these values should always be positive. They are signed only to retain consistency with PALib's Stlyus struct.

To add a window to our screen:

// Create window
Window* window = screen->addWindow(20, 20, 100, 100, "My Window", false, true);

This will create a window at (20, 20) and return a pointer to it, which we will use to add gadgets to the window.

Creating a Gadget

You can probably see a pattern emerging. Each new item we create is added to the previous item. What we are doing is building a hierarchy of GUI objects - windows belong to screens, which belong to Woopsi.

Gadgets follow the same pattern. There are several classes of gadget that can be added to windows, and each offers different functionality. The classes of gadget that exist at the moment are:

  • Textbox - a box that will display a single line of text
  • Button - a clickable box that displays a single line of text and that will fire an event when clicked
  • TextViewer - a gadget that displays multiple lines of text (as many as the DS' memory will allow) that can be scrolled using the stylus
  • SuperBitmap - provides canvas on which we can draw using a set of drawing functions; the canvas can be larger than the window, and can be scrolled with the stylus

If we want to add a button to our window, we would use this function:

Button* Window::addButton(s16 x, s16 y, u16 width, u16 height, char* text)

The parameters follow the same pattern as the addWindow() function above. To add a button, we use:

// Create button
Button* button = window->addButton(5, 5, 60, 60, "Hello World!"));

Summary

In four lines of code, we've created a Woopsi instance, added a screen to it, created a window and defined a clickable button. Extracting all of the code from the above into an easily-viewable format gives us this:

// Create a simple GUI
Woopsi woopsi;
Screen* screen = woopsi.addScreen("My Screen");
Window* window = screen->addWindow(20, 20, 100, 100, "My Window", false, true);
Button* button = window->addButton(5, 5, 60, 60, "Hello World!"));
